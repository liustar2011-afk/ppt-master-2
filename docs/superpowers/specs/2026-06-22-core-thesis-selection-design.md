# 核心论点选择交互设计

确认页将核心论点从单一自由输入改为候选句选择。策略角色在 `recommendations.json.core_thesis.candidates` 中提供两至三个、基于同一SCQA和支撑论点的候选句，并通过 `selected` 标记推荐项。用户可选择任一候选句，或进入自定义输入；提交时核心论点必须为非空文本。

为保持既有项目可用，前端继续读取旧的 `core_thesis.thesis` 字段：当不存在候选数组时，将它转换为唯一候选。回写协议保持 `result.json.core_thesis` 为最终字符串，后续策略与执行流程不需要变更。

改动范围限于确认页前端、确认页协议文档以及策略角色对推荐数据的生成要求。无需修改服务端持久化逻辑、PPT生成链路或项目文件格式。
